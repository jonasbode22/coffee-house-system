# Arquitetura do Sistema

O Sistema de Gestão para Cafeteria foi organizado utilizando uma arquitetura em camadas para facilitar a manutenção, organização e evolução do projeto.

## Componentes

- Interface do Usuário (Frontend)
- Lógica de Negócio (Backend)
- Banco de Dados

## Fluxo Geral

Usuário

↓

Frontend

↓

Backend

↓

Banco de Dados

↓

Resposta ao Usuário

## Benefícios

- Organização do código.
- Facilidade de manutenção.
- Reutilização de componentes.
- Escalabilidade do sistema.
